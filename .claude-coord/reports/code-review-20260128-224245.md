## Critical Issues (MUST FIX BEFORE PRODUCTION)

### Security Vulnerabilities

#### 1. **SSRF Vulnerability - DNS Rebinding Attack** (tools/web_scraper.py)
- **Module:** tools
- **Risk:** Attackers can bypass SSRF protections via DNS rebinding, accessing internal services and cloud metadata endpoints
- **Fix:** Validate resolved IPs AFTER connection establishment using transport hooks

#### 2. **Command Injection - Incomplete Sanitization** (tools/base.py)
- **Module:** tools
- **Risk:** Bypass shell metacharacter filtering using Unicode homoglyphs, command substitution, brace expansion
- **Fix:** Add Unicode normalization, expand dangerous pattern list, enforce subprocess with shell=False

#### 3. **Path Traversal - Multiple Bypass Techniques** (tools/base.py, safety/rollback.py, core/cli/)
- **Module:** Multiple
- **Risk:** Encoded paths (%2e%2e), symlink attacks, TOCTOU race conditions allow arbitrary file access
- **Fix:** Decode URL encoding, handle symlinks, use content hashing, centralize validation

#### 4. **Cache Key Collision - Response Poisoning** (cache/llm_cache.py)
- **Module:** cache
- **Risk:** Different prompts with varying system_prompt/tools generate same cache key, returning wrong responses
- **Fix:** Include ALL request parameters (system_prompt, tools) in cache key hash

#### 5. **Template Injection in Jinja2** (agents/prompt_engine.py)
- **Module:** agents
- **Risk:** SSTI attacks can bypass SandboxedEnvironment, accessing Python internals and executing code
- **Fix:** Whitelist template variables, validate types, consider string.Template for untrusted input

#### 6. **Arbitrary Code Execution via Dynamic Tool Loading** (agents/standard_agent.py)
- **Module:** agents
- **Risk:** Malicious tool classes could be dynamically imported if whitelist is compromised
- **Fix:** Strict regex validation, plugin signatures, sandboxed execution, audit logging

#### 7. **Secret Detection ReDoS** (safety/secret_detection.py)
- **Module:** safety
- **Risk:** Regex catastrophic backtracking causes DoS on crafted input
- **Fix:** Use atomic groups, timeout support, more specific character classes

#### 8. **Sensitive Data Logging** (observability/tracker.py)
- **Module:** observability
- **Risk:** Full LLM prompts/responses logged without sanitization, exposing API keys, PII, credentials
- **Fix:** Implement prompt sanitization, PII detection, content redaction before storage
