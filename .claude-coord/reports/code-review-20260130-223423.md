## Critical Issues (P0 - Fix Immediately)

### Security Vulnerabilities

#### 1. **SQL Injection via Raw Migration Execution** (observability)
- **Location:** `src/observability/migrations.py:99-126`
- **Risk:** Database compromise, data loss
- **Issue:** Raw SQL migrations executed without proper validation. Uppercase-only pattern checking easily bypassed.
- **Recommendation:** Replace with Alembic framework or implement cryptographic signing of migration files.

#### 2. **ReDoS Vulnerability in Forbidden Operations** (safety)
- **Location:** `src/safety/forbidden_operations.py:96`
- **Risk:** DoS via CPU exhaustion
- **Issue:** Nested quantifiers in regex: `[^|]*\s*>\s*[^&>\s|]+\.(txt|json|...)`
- **Attack:** Input like `"echo " + "a" * 10000 + " >"` causes catastrophic backtracking
- **Fix:** Use possessive quantifiers or atomic groups

#### 3. **Path Traversal via Rollback Operations** (safety)
- **Location:** `src/safety/rollback.py:217-231`
- **Risk:** Read arbitrary system files
- **Issue:** File snapshot creation doesn't validate paths, allows symlink following
- **Fix:** Validate with `os.path.realpath()` and check against allowed directories

#### 4. **Arbitrary File Deletion in Rollback** (safety)
- **Location:** `src/safety/rollback.py:264-273`
- **Risk:** System compromise, data loss
- **Issue:** Deletes files based on metadata without path validation
- **Fix:** Whitelist allowed directories, add ownership checks

#### 5. **Insecure Cache Key Generation** (safety)
- **Location:** `src/safety/action_policy_engine.py:287-309`
- **Risk:** Cache poisoning, policy bypass
- **Issue:** JSON serialization not deterministic, allows cache collision attacks
- **Fix:** Use canonical representation or HMAC-based keys

#### 6. **Sensitive Data Exposure in Logs** (safety + observability)
- **Locations:**
  - `src/safety/secret_detection.py:222`
  - `src/observability/tracker.py:478-546`
- **Risk:** GDPR/CCPA violations, credential exposure
- **Issue:** Full LLM prompts/responses and detected secrets logged without sanitization
- **Fix:** Implement redaction patterns, PII detection, encryption at rest

#### 7. **Command Injection via Environment Variables** (compiler)
- **Location:** `src/compiler/config_loader.py:556-565`
- **Risk:** Remote code execution
- **Issue:** Validation only checks variable names matching patterns ('cmd', 'command'), easily bypassed
- **Fix:** Validate ALL environment variable values, whitelist safe characters

#### 8. **SQL Injection Pattern Detection Incomplete** (compiler)
- **Location:** `src/compiler/config_loader.py:568-588`
- **Risk:** SQL injection bypass
- **Issue:** Detection only checks uppercase patterns, bypassed with mixed case
- **Fix:** Case-insensitive checking, require parameterized queries at framework level

#### 9. **SSRF DNS Timing Attack** (tools)
- **Location:** `src/tools/web_scraper.py:40-105`
- **Risk:** Internal network reconnaissance, DNS rebinding
- **Issue:** DNS resolution on attacker-controlled hostnames without timeout
- **Fix:** Add DNS resolution timeout (2s), implement DNS caching

#### 10. **AST-based Calculator DoS** (tools)
- **Location:** `src/tools/calculator.py:148-236`
- **Risk:** Stack overflow, memory exhaustion
- **Issue:** Deeply nested lists `[[[[...]]]]` cause unbounded recursion
- **Fix:** Add depth tracking with max_depth=10 limit

#### 11. **Path Traversal via Symlinks** (tools)
- **Location:** `src/tools/file_writer.py:136-148`
- **Risk:** Write to arbitrary system paths
- **Issue:** `Path.resolve()` follows symlinks, bypasses directory restrictions
- **Fix:** Check for symlinks before resolution

#### 12. **Weak Cryptography in Secrets Module** (utils)
- **Location:** `src/utils/secrets.py:198`
- **Risk:** False sense of security
- **Issue:** Fernet with in-memory keys provides no real protection
- **Fix:** Use OS keyring or document as obfuscation, not encryption

#### 13. **Path Traversal in /tmp Access** (utils)
- **Location:** `src/utils/path_safety.py:142-156`
- **Risk:** Access other users' files, symlink attacks
- **Issue:** Allows `/tmp` access unconditionally without validation
- **Fix:** Remove exception or add ownership checks, symlink detection

#### 14. **ReDoS in Secret Detection** (utils)
- **Location:** `src/utils/secrets.py:285-310`
- **Risk:** CPU exhaustion DoS
- **Issue:** Pattern `[A-Za-z0-9+/]{40,}={0,2}` vulnerable to catastrophic backtracking
- **Fix:** Add regex timeout, use non-backtracking patterns

#### 15. **Cache Poisoning via Key Injection** (cache)
- **Location:** `src/cache/llm_cache.py:362-378`
- **Risk:** Cache poisoning, response manipulation
- **Issue:** `json.dumps()` doesn't sanitize kwargs, allows key collisions
- **Fix:** Whitelist allowed kwargs or use structured hashing

#### 16. **Hash Collision Privacy Leak** (cache)
- **Location:** `src/cache/llm_cache.py:374-375`
- **Risk:** Cross-tenant data leakage
- **Issue:** No user_id/tenant_id in hash, identical prompts share cache
- **Fix:** Include user context in hash or use HMAC with secret

#### 17. **Circuit Breaker State Race Condition** (llm)
- **Location:** `src/llm/circuit_breaker.py:100-111`
- **Risk:** Execution during circuit open, service overload
- **Issue:** Check state → release lock → execute → acquire lock. State can change mid-flight.
- **Fix:** Hold lock through entire execution or use double-checked locking

#### 18. **ReDoS in Prompt Injection Detection** (security)
- **Location:** `src/security/llm_security.py:59-93`
- **Risk:** DoS via regex backtracking
- **Issue:** Pattern with nested quantifiers: `ignore[\s._\-|]+(all[\s._\-|]+)?previous[\s._\-|]+instructions`
- **Attack:** `"ignore all previous........[thousands of dots]"` causes exponential execution
- **Fix:** Use atomic groups, add regex timeout

#### 19. **Entropy Calculation DoS** (security)
- **Location:** `src/security/llm_security.py:156-174`
- **Risk:** Memory exhaustion
- **Issue:** 10MB Unicode input creates massive character dictionary
- **Fix:** Add length limit (100KB) before entropy calculation

#### 20. **Secret Sanitization Bypass** (security)
- **Location:** `src/security/llm_security.py:268-301`
- **Risk:** Partial secret leakage
- **Issue:** Overlapping secret patterns processed in reverse, shorter match may leak data
- **Fix:** Use longest-match-first strategy or span merging

#### 21. **SQL Injection via Experiment Name** (experimentation)
- **Location:** `src/experimentation/service.py:86-181`
- **Risk:** Data breach via timing attacks
- **Issue:** Name parameter not sanitized before database operations
- **Fix:** Validate name matches `^[a-zA-Z0-9_-]{1,50}$` pattern

   - **Fix:** Force synchronous close or track cleanup tasks

   - Sequential policy iteration causes multiple DB queries
   - **Impact:** Slow validation, database load
   - **Fix:** Batch policy loading or implement caching
