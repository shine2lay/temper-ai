# Contributors

Thank you to everyone who has contributed to the Meta-Autonomous Framework!

## Core Team

This project is actively developed and maintained by the Meta-Autonomous Framework team.

## How to Contribute

We welcome contributions! Please see our [Contributing Guide](./docs/CONTRIBUTING.md) for details on:

- Code of conduct
- Development setup
- Coding standards
- Testing requirements
- Pull request process
- Documentation guidelines

## Types of Contributions

We appreciate all types of contributions, including but not limited to:

### Code Contributions
- Feature implementations
- Bug fixes
- Performance improvements
- Test coverage enhancements
- Code refactoring

### Documentation
- API documentation
- Usage examples
- Architecture guides
- Tutorial content
- Documentation fixes

### Testing
- Test case development
- Integration tests
- Performance benchmarks
- Edge case coverage
- Bug reports with reproduction steps

### Design & Architecture
- Architecture reviews
- Design proposals
- ADR (Architecture Decision Records) contributions
- System design feedback

### Community
- Issue triage
- Question answering
- Community support
- Educational content
- Use case sharing

## Recognition

Contributors are recognized in several ways:

- Listed in this CONTRIBUTORS.md file
- Mentioned in relevant CHANGELOG.md entries
- Referenced in commit messages with Co-Authored-By
- Acknowledged in milestone completion reports

## Contribution Statistics

To see contribution statistics:
```bash
# View commit authors
git shortlog -sn --all --no-merges

# View contributors by lines of code
git log --format='%aN' | sort -u

# View recent contributors
git log --since="1 month ago" --format='%aN' | sort -u
```

## Getting Started

1. **Fork the repository**
2. **Read** [CONTRIBUTING.md](./docs/CONTRIBUTING.md)
3. **Set up** your development environment
4. **Pick an issue** labeled `good-first-issue` or `help-wanted`
5. **Submit** your pull request

## Communication

- **Issues:** Report bugs and request features via GitHub Issues
- **Pull Requests:** Submit code changes via GitHub Pull Requests
- **Discussions:** Ask questions and share ideas in GitHub Discussions
