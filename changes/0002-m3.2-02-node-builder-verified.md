## Architecture

### Component Hierarchy

```
LangGraphCompiler (Orchestrator - 198 lines)

    ├── StateManager (State initialization)
    ├── NodeBuilder (Node creation) ← EXTRACTED
```

### NodeBuilder Responsibilities

1. **Create Stage Execution Nodes** (`create_stage_node()`)
   - Returns callable functions for LangGraph StateGraph
   - Wraps executor delegation in LangGraph-compatible interface
   - Converts state between LangGraph and executor formats

2. **Load Stage Configurations** (`_load_stage_config()`)
   - Load from file-based configs via ConfigLoader
   - Fallback to embedded stage search (future enhancement)
   - Proper error handling with ValueError

3. **Determine Execution Mode** (`get_agent_mode()`)
   - Extract from Pydantic models or dict configs
   - Supports: sequential, parallel, adaptive
   - Defaults to "sequential"

4. **Extract Names** (utility methods)
   - `extract_stage_name()`: From string/dict/Pydantic → stage name
   - `extract_agent_name()`: From string/dict/Pydantic → agent name
   - Handles multiple key variations (name, stage_name, agent_name, etc.)

## Key Features Implemented

### 1. Stateless Design
- No internal state (all data passed as parameters)
- Thread-safe by design
- Easy to test with mocks

### 2. Multiple Config Format Support
- **Pydantic models**: `stage_config.stage.execution.agent_mode`
- **Dictionaries**: `stage_config["execution"]["agent_mode"]`
- **Strings**: Direct stage/agent name references
- Backward compatible with existing configs

### 3. Executor Delegation Pattern
- NodeBuilder creates nodes, Executors handle execution
- Clean separation of concerns
- Enables multiple execution strategies (sequential/parallel/adaptive)

### 4. LangGraph Integration
- Returns LangGraph-compatible callable nodes
- Proper state conversion (LangGraphWorkflowState ↔ dict)
- Returns dict updates (not full state) per LangGraph best practice

### 5. Error Handling
- `ValueError` for config loading failures
- `ValueError` for name extraction failures
- Clear error messages with context

---

## Design Decisions

### 1. Stateless vs Stateful
**Decision**: Stateless (no config caching in NodeBuilder)
**Reasoning**: ConfigLoader already handles caching if needed, simpler design

### 2. Executor Delegation
**Decision**: Delegate execution to specialized executor classes
**Reasoning**: Separation of concerns, enables multiple strategies, better testability

### 3. Support Multiple Config Formats
**Decision**: Support both Pydantic and dict formats
**Reasoning**: Backward compatibility + future type safety

### 4. Return Dict Updates
**Decision**: Stage nodes return dict updates, not full state
**Reasoning**: LangGraph best practice, more efficient, clearer intent
