## Summary

Extracted node creation logic from `LangGraphCompiler` into dedicated `NodeBuilder` class, reducing compiler complexity and centralizing graph node construction responsibilities.

## Motivation

The `LangGraphCompiler` was handling too many responsibilities:
- Compiling workflows into graphs
- Creating individual stage nodes
- Extracting stage/agent names from various formats
- Determining agent execution modes
- Managing executor delegation

This refactoring extracts node building logic into a focused class, following Single Responsibility Principle.

## Changes Made

- Centralizes all node creation logic
- Manages executor delegation
- Handles configuration format variations

## Migration Guide

### For Test Writers

**Before:**
```python
compiler._create_stage_node("research", workflow_config)
compiler._get_agent_mode(stage_config)
compiler._extract_stage_name(stage_ref)
```

**After:**
```python
compiler.node_builder.create_stage_node("research", workflow_config)
compiler.node_builder.get_agent_mode(stage_config)
compiler.node_builder.extract_stage_name(stage_ref)
```

### For Executor Developers

Node creation is now fully centralized in `NodeBuilder.create_stage_node()`. To add support for a new executor:

1. Register executor in `LangGraphCompiler.__init__()`:
   ```python
   self.executors['my_mode'] = MyStageExecutor()
   ```

2. NodeBuilder will automatically delegate based on `stage_config.execution.agent_mode`

No changes needed to NodeBuilder itself!
