## Summary

Extracted LangGraph-specific graph construction logic from `LangGraphCompiler` into dedicated `StageCompiler` class, further reducing compiler complexity and isolating LangGraph StateGraph concerns.

## Motivation

After extracting StateManager (m3.2-01) and NodeBuilder (m3.2-02), the LangGraphCompiler still contained graph construction logic:
- Creating StateGraph instances
- Adding initialization and stage nodes
- Connecting nodes with edges
- Setting entry points
- Compiling graphs

This refactoring extracts these LangGraph-specific concerns into StageCompiler, leaving LangGraphCompiler as a high-level orchestrator.

### 2. Refactored `src/compiler/langgraph_compiler.py`

**Simplified `compile()` method:**

**Changes:**
```python
# Before (52 lines):
def compile(self, workflow_config):
    workflow = workflow_config.get("workflow", workflow_config)
    stages = workflow.get("stages", [])
    if not stages:
        raise ValueError("Workflow must have at least one stage")

    graph = StateGraph(WorkflowState)
    stage_names = [self._extract_stage_name(stage) for stage in stages]

    graph.add_node("init", self._create_start_node())
    for stage_name in stage_names:
        stage_node = self._create_stage_node(stage_name, workflow_config)
        graph.add_node(stage_name, stage_node)

    graph.add_edge(START, "init")
    graph.add_edge("init", stage_names[0])
    for i in range(len(stage_names) - 1):
        graph.add_edge(stage_names[i], stage_names[i + 1])
    graph.add_edge(stage_names[-1], END)

    graph.set_entry_point("init")
    return graph.compile()

# After (14 lines):
def compile(self, workflow_config):
    """Orchestrate compilation: parse config, extract names, delegate to StageCompiler."""
    workflow = workflow_config.get("workflow", workflow_config)
    stages = workflow.get("stages", [])

    if not stages:
        raise ValueError("Workflow must have at least one stage")

    stage_names = [
        self.node_builder.extract_stage_name(stage)
        for stage in stages
    ]

    return self.stage_compiler.compile_stages(stage_names, workflow_config)
```

### For Framework Developers
To add new graph construction patterns:

1. Add method to `StageCompiler`:
   ```python
   def compile_custom_pattern(self, stage_names, workflow_config, ...):
       graph = StateGraph(WorkflowState)
       # Custom graph construction logic
       return graph.compile()
   ```

2. Call from `LangGraphCompiler.compile()`:
   ```python
   if workflow.get("execution_mode") == "custom":
       return self.stage_compiler.compile_custom_pattern(...)
   ```
