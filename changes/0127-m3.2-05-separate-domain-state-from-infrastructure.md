# Change Log: m3.2-05 - Separate Domain State from Infrastructure

**Date:** 2026-01-27
**Task ID:** m3.2-05
**Status:** Completed
**Author:** Claude (Sonnet 4.5)

## Summary

Separated workflow state into two distinct layers: WorkflowDomainState (pure serializable domain data) and ExecutionContext (infrastructure components). This separation is the foundation for checkpoint/resume capability (m3.2-06) and follows best practices for state management.

## Motivation

The original `WorkflowState` class mixed two fundamentally different concerns:
1. **Domain State**: Serializable workflow data (inputs, outputs, progress)
2. **Infrastructure**: Non-serializable runtime components (trackers, registries)

This mixing prevented checkpoint/resume because:
- Can't serialize infrastructure objects (trackers, registries contain connections, state)
- No clear boundary between "what to save" vs "what to recreate"
- Tightly coupled domain logic with infrastructure dependencies

## Solution

### Created Clean Separation

```python
# Domain State (SERIALIZABLE - can be checkpointed)
@dataclass
class WorkflowDomainState:
    stage_outputs: Dict[str, Any]  # Workflow progress
    current_stage: str             # Current position
    workflow_id: str               # Execution ID
    topic: Optional[str]           # User inputs
    input: Optional[str]
    # ... all JSON-serializable fields

# Infrastructure (NON-SERIALIZABLE - recreated on resume)
@dataclass
class ExecutionContext:
    tracker: Optional[Any]         # Runtime observability
    tool_registry: Optional[Any]   # Runtime tool access
    config_loader: Optional[Any]   # Runtime config loading
    visualizer: Optional[Any]      # Runtime visualization
```

### Checkpoint/Resume Pattern

```python
# Save checkpoint
domain = WorkflowDomainState(workflow_id="wf-123", input="data")
domain.set_stage_output("research", {"findings": [...]})
checkpoint = domain.to_dict()  # Only domain data
json.dump(checkpoint, file)

# Resume from checkpoint
domain = WorkflowDomainState.from_dict(json.load(file))
context = ExecutionContext(  # Recreate infrastructure
    tracker=ExecutionTracker(),
    tool_registry=ToolRegistry()
)
```

### 1. Created `src/compiler/domain_state.py` (409 lines)

**New Classes:**

#### `WorkflowDomainState`
- **Purpose**: Pure serializable domain state
- **Fields**: All JSON-serializable workflow data
- **Key Methods**:
  - `to_dict()`: Serialize to JSON-safe dict
  - `from_dict()`: Deserialize from checkpoint
  - `validate()`: Validate state consistency
  - `copy()`: Deep copy for independent state

```python
@dataclass
class WorkflowDomainState:
    # Core workflow state
    stage_outputs: Dict[str, Any] = field(default_factory=dict)
    current_stage: str = ""
    workflow_id: str = field(default_factory=lambda: f"wf-{uuid.uuid4().hex[:12]}")

    # Workflow inputs (all optional)
    topic: Optional[str] = None
    depth: Optional[str] = None
    focus_areas: Optional[List[str]] = None
    query: Optional[str] = None
    input: Optional[str] = None
    context: Optional[str] = None
    data: Optional[Any] = None

    # Metadata
    version: str = "1.0"
    created_at: datetime = field(default_factory=datetime.utcnow)
    metadata: Dict[str, Any] = field(default_factory=dict)
```

#### `ExecutionContext`
- **Purpose**: Infrastructure component container
- **Fields**: All non-serializable runtime components
- **Design**: All fields optional, not checkpointed

```python
@dataclass

    tracker: Optional[Any] = None
    tool_registry: Optional[Any] = None
    config_loader: Optional[Any] = None
    visualizer: Optional[Any] = None
```

**Helper Functions:**

### 2. Modified `src/compiler/state.py` (now compatibility facade)

**Status**: Now deprecated facade over separated architecture

**Changes:**
- Added deprecation warnings
- Delegates all operations to `WorkflowDomainState` and `ExecutionContext`
- Maintains backward compatibility
- Separates constructor kwargs into domain vs context fields

```python
class WorkflowState:
    """DEPRECATED: Compatibility facade over separated architecture.

    New code should use:
    - WorkflowDomainState for domain state
    - ExecutionContext for infrastructure
    """

    def __init__(self, **kwargs: Any):
        # Separate domain and context fields
        domain_fields = {...}
        context_fields = {...}

        # Create separated components
        self._domain = WorkflowDomainState(**domain_fields)
        self._context = ExecutionContext(**context_fields)

    # Delegate all operations to separated components
    def set_stage_output(self, stage_name: str, output: Any):
        self._domain.set_stage_output(stage_name, output)
```

## Architecture Impact

### Before (M3.2-04)
```
WorkflowState
├── stage_outputs (domain)
├── current_stage (domain)
├── workflow_id (domain)
├── input (domain)
├── tracker (infrastructure) ❌ Mixed!
├── tool_registry (infrastructure) ❌ Mixed!
└── config_loader (infrastructure) ❌ Mixed!
```

### After (M3.2-05)
```
WorkflowDomainState (SERIALIZABLE)
├── stage_outputs
├── current_stage
├── workflow_id
├── input
├── topic
├── depth
└── ... (all JSON-serializable)

ExecutionContext (NON-SERIALIZABLE)
├── tracker
├── tool_registry
├── config_loader
└── visualizer
```

## Benefits

1. **Checkpoint/Resume Foundation**: Clear separation enables saving/loading state
2. **Serialization Safety**: Domain state guaranteed JSON-serializable
3. **Infrastructure Flexibility**: Can recreate context with different implementations
4. **Testing**: Can test domain logic without infrastructure dependencies
5. **State Migrations**: Version field enables schema evolution
6. **Clarity**: Explicit boundary between domain and infrastructure
7. **Backward Compatibility**: Existing code continues to work via facade

### Old Code (Deprecated but still works)

### New Code (Recommended)
```python

# Separate domain and infrastructure
domain = WorkflowDomainState(
    workflow_id="wf-123",
    input="analyze data"
)

context = ExecutionContext(
    tracker=my_tracker,
    tool_registry=my_registry
)
```

### Checkpoint Usage
```python
# Save checkpoint
checkpoint_data = domain.to_dict()
with open("checkpoint.json", "w") as f:
    json.dump(checkpoint_data, f)

# Resume from checkpoint
with open("checkpoint.json", "r") as f:
    checkpoint_data = json.load(f)

domain = WorkflowDomainState.from_dict(checkpoint_data)

# Recreate infrastructure (not from checkpoint)
context = ExecutionContext(
    tracker=ExecutionTracker(),
    tool_registry=ToolRegistry()
)
```

## Design Principles

### 1. Clear Separation of Concerns
- **Domain State**: Business logic data only
- **Infrastructure**: Runtime components only
- **No mixing**: Strict boundary enforcement

### 2. Serialization by Design
- All domain fields JSON-serializable
- Infrastructure explicitly excluded from serialization
- Datetime handling for ISO format

### 3. Validation and Safety
- Automatic workflow_id prefixing
- Type validation in `__post_init__`
- Explicit `validate()` method
- Deep copy support for state independence

### 4. Version Management
- Version field for schema evolution
- `from_dict()` handles version differences
- Extra fields preserved in metadata
- Forward/backward compatibility support
